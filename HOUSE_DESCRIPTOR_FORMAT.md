# Спецификация формата JSON-дескриптора дома

**Версия формата**: 1.0
**Статус**: согласовано
**Файлы дескрипторов**: `assets/houses/house_type_*.json`

---

## 1. Обзор

Каждый тип дома описывается одним JSON-файлом. Файл определяет:

- Метаданные (название, описание)
- Ограничения параметров (допустимые диапазоны площади, высоты и т.д.)
- Тип крыши
- Этажи (один или несколько), каждый со своей планировкой
- Планировку каждого этажа — как **обход периметра по часовой стрелке**
- Дополнительные элементы (дымоход, водостоки, карниз)
- Карту заменяемых материалов

---

## 2. Корневая структура

```jsonc
{
  // ── Метаданные ──────────────────────────────────────
  "id":          "type_a",                              // string, уникальный ID
  "name":        "Одноэтажный дом",                     // string, отображается в UI
  "description": "С вальмовой крышей",                  // string, подпись

  // ── Общие ограничения (для всего дома) ──────────────
  "constraints": {
    "base_h":          { "min": 50, "max": 120, "step": 10, "default": 80, "unit": "cm" },
    "roof_angle":      { "min": 15, "max": 45,  "step": 5,  "default": 25, "unit": "deg" },
    "wall_thickness":  0.20   // метры, фиксировано
  },

  // ── Тип крыши ───────────────────────────────────────
  "roof_type": "hip",         // "hip" | "gable" | "flat"

  // ── Каталог модулей (размеры проёмов) ──────────────
  "modules": { /* ... см. раздел 2.1 ... */ },

  // ── Этажи ───────────────────────────────────────────
  "floors": [ /* ... см. раздел 3 ... */ ],

  // ── Доп. элементы ───────────────────────────────────
  "features": { /* ... см. раздел 7 ... */ },

  // ── Карта материалов ────────────────────────────────
  "materials_map": { /* ... см. раздел 8 ... */ }
}
```

### Обязательные поля верхнего уровня

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | string | Уникальный идентификатор типа дома |
| `name` | string | Название для отображения в UI |
| `constraints` | object | Общие ограничения (фундамент, крыша, толщина стены) |
| `roof_type` | string | Тип крыши: `"hip"`, `"gable"`, `"flat"` |
| `modules` | object | Каталог модулей с размерами проёмов (см. раздел 2.1) |
| `floors` | array | Массив этажей (минимум 1) |
| `materials_map` | object | Перечень заменяемых материалов |

### Необязательные поля

| Поле | Тип | Описание |
|------|-----|----------|
| `description` | string | Подпись в UI |
| `features` | object | Дымоход, водостоки, карниз |

### 2.1. Каталог модулей (`modules`)

Секция `modules` задаёт размеры всех оконных и дверных модулей, используемых
в `perimeter.facade`. Код использует эти значения для расчёта `fill`-заполнителей
и позиционирования проёмов по высоте.

```jsonc
"modules": {
  "window_single": { "w": 0.90, "h": 1.20, "y": 0.90,  "glb": "mod_window_single.glb" },
  "window_double": { "w": 1.40, "h": 1.20, "y": 0.90,  "glb": "mod_window_double.glb" },
  "window_wide":   { "w": 2.40, "h": 1.50, "y": 0.60,  "glb": "mod_window_wide.glb"   },
  "door_entrance": { "w": 1.00, "h": 2.20, "y": 0.00,  "glb": "mod_door_entrance.glb" },
  "door_patio":    { "w": 2.00, "h": 2.30, "y": 0.00,  "glb": "mod_door_patio.glb"    }
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `w` | number | Ширина проёма (включая раму), метры. Используется при расчёте `fill` |
| `h` | number | Высота проёма (включая раму), метры |
| `y` | number | Расстояние от уровня пола этажа до **нижнего края** проёма, метры |
| `glb` | string | Имя GLB-файла в `assets/modules/windows/` или `assets/modules/doors/` |

**Ключ** объекта (`window_single`, `door_entrance` и т.д.) — это ID модуля,
на который ссылаются элементы фасада: `{ "window": "single" }` → ищет `window_single`.

**Правила формирования ключа**: `<тип>_<вариант>`, где тип = `window` | `door`.
В элементе фасада пишется только вариант: `{ "window": "single" }`, `{ "door": "entrance" }`.

Новые типы окон/дверей добавляются расширением этой секции:
```jsonc
"window_arch":     { "w": 1.00, "h": 1.60, "y": 0.80, "glb": "mod_window_arch.glb" },
"door_garage":     { "w": 3.00, "h": 2.50, "y": 0.00, "glb": "mod_door_garage.glb" }
```

## 3. Этаж (`floors[i]`)

Каждый элемент массива `floors` описывает один этаж:

```jsonc
{
  "id":       "floor_1",           // string, уникальный ID этажа
  "label":    "Первый этаж",       // string, отображается в UI

  "constraints": {
    "area":    { "min": 40, "max": 100, "step": 5,  "default": 80,  "unit": "m2" },
    "floor_h": { "min": 270, "max": 360, "step": 10, "default": 300, "unit": "cm" }
  },

  "vars": {
    "L": "sqrt(area * 1.6)",
    "W": "sqrt(area / 1.6)"
  },

  "perimeter": [ /* ... см. раздел 4 ... */ ]
}
```

### Поля этажа

| Поле | Тип | Обязательное | Описание |
|------|-----|:---:|----------|
| `id` | string | ✓ | Уникальный ID, используется в `S.wallZones` |
| `label` | string | ✓ | Название для UI |
| `constraints` | object | ✓ | Допустимые значения площади и высоты этажа |
| `vars` | object | ✓ | Параметрические переменные (см. раздел 5) |
| `perimeter` | array | ✓ | Обход периметра (см. раздел 4) |

### Ограничения этажа (`constraints`)

| Ключ | Что задаёт | Единицы |
|------|-----------|---------|
| `area` | Площадь этажа в плане | м² |
| `floor_h` | Высота этажа (от пола до потолка) | см |

Каждое ограничение — объект:
```jsonc
{ "min": 40, "max": 100, "step": 5, "default": 80, "unit": "m2" }
```

Код конфигуратора:
- Устанавливает `min`, `max`, `step` на элементы ввода
- Округляет значения до шага: `value = round(raw / step) * step`
- Не допускает выход за диапазон

### Многоэтажность

- Этажи собираются **снизу вверх** (первый элемент `floors` = нижний этаж)
- Между этажами автоматически добавляется **перекрытие** толщиной 0.2 м
- Каждый этаж может иметь **свою площадь и пропорции**
- Если верхний этаж меньше нижнего — образуется уступ (пользователь может вручную разметить на нём балкон)
- Верхний этаж **автоматически центрируется** относительно нижнего

## 4. Обход периметра (`perimeter`)

Планировка задаётся как **обход контура по часовой стрелке** (turtle graphics).

### 4.1. Начальные условия

- Стартовая точка: `[0, 0]` (верхний-левый угол на плане)
- Начальное направление: **вправо** (+X)
- Система координат: X — вправо, Z — вниз (вид сверху)

### 4.2. Типы команд

Массив `perimeter` содержит чередующиеся команды двух типов:

#### Команда `run` — движение и рисование стены

```jsonc
{
  "run": "L",              // string — имя переменной из vars (длина участка)
  "facade": [ /* ... */ ]  // array | string — содержимое стены (см. раздел 6)
}
```

#### Команда `turn` — поворот

```jsonc
{ "turn": 90 }   // число — угол поворота в градусах
```

| Значение | Направление | Эффект на контуре |
|----------|-------------|-------------------|
| `90` | По часовой стрелке | Обычный угол (внутренний угол дома) |
| `-90` | Против часовой стрелки | Создаёт **выступ** (внешний угол дома) |
| `180` | Разворот | Очень редко, для U-образных элементов |

### 4.3. Правило замкнутости

Контур **обязан быть замкнутым**: после выполнения всех команд текущая точка должна
совпасть с начальной `[0, 0]`. Код проверяет это и выдаёт предупреждение при ошибке.

### 4.4. Порядок команд

Команды `run` и `turn` обычно чередуются:
```
run → turn → run → turn → run → turn → run → turn   (прямоугольник: 4 run + 4 turn)
```

Для сложных планов (L, T, крест) количество команд больше, но принцип тот же.

### 4.5. Комментарии

JSON не поддерживает комментарии, но можно использовать `"_comment"`:
```jsonc
{ "_comment": "── Северная стена →" }
```
Код игнорирует элементы без `run` и без `turn`.

## 5. Параметрические переменные (`vars`)

### 5.1. Формат

```jsonc
"vars": {
  "L":    "sqrt(area * 1.6)",      // длина дома
  "W":    "sqrt(area / 1.6)",      // ширина дома
  "wing": "W * 0.3",              // ширина крыла
  "arm":  "L * 0.25"              // длина крыла
}
```

Каждая переменная — это строка-выражение, которое вычисляется в числовое значение (метры).

### 5.2. Доступные в выражениях величины

| Имя | Откуда | Описание |
|-----|--------|----------|
| `area` | Из UI (constraints этажа) | Площадь этажа, м² |
| Другие переменные из `vars` | Этот же блок | Можно ссылаться на уже определённые |

### 5.3. Поддерживаемые операции

| Операция | Пример | Описание |
|----------|--------|----------|
| `+` `-` `*` `/` | `"area * 1.6"` | Арифметика |
| `sqrt()` | `"sqrt(area * 1.6)"` | Квадратный корень |
| `()` | `"(L + W) / 2"` | Группировка |
| Числовые литералы | `"1.6"`, `"0.3"` | Константы |

### 5.4. Порядок вычисления

Переменные вычисляются **сверху вниз** — каждая может ссылаться только на уже
определённые выше переменные и на `area`.

```jsonc
"vars": {
  "L":    "sqrt(area * 1.6)",    // OK: использует area
  "W":    "sqrt(area / 1.6)",    // OK: использует area
  "wing": "W * 0.3",            // OK: W уже определена
  "diag": "sqrt(L*L + W*W)"     // OK: L и W уже определены
}
```

### 5.5. Когда пересчитываются

Переменные пересчитываются при каждом изменении параметров пользователем
(ползунок площади, поле высоты и т.д.). После пересчёта контур перестраивается.

## 6. Фасад (`facade`)

Описывает содержимое стены на одном `run` — что расположено на этом участке
**слева направо по ходу движения**.

### 6.1. Формат — массив элементов

```jsonc
"facade": [
  { "wall": 1.2 },            // глухая стена 1.2 м
  { "window": "single" },     // одностворчатое окно
  { "wall": "fill" },         // заполнитель
  { "window": "single" },     // ещё окно
  { "wall": 1.2 }             // глухая стена 1.2 м
]
```

### 6.2. Формат — автоматические окна

```jsonc
"facade": "auto_windows",     // строка вместо массива
"window_type": "single",      // какой тип окна расставлять
"min_margin": 1.0             // мин. отступ от краёв и между окнами (по умолчанию 1.0)
```

Код рассчитает:
```
winCount = floor(runLength / (windowWidth + min_margin * 2))
margin = (runLength - winCount * windowWidth) / (winCount + 1)
```

### 6.3. Типы элементов фасада

#### Глухая стена

```jsonc
{ "wall": 2.0 }       // фиксированная длина, метры
{ "wall": "fill" }     // заполнитель (вычисляется)
```

**`"fill"`** — оставшееся после размещения фиксированных элементов место делится
**поровну** между всеми `"fill"` в этом `run`.

Алгоритм:
```
totalRun  = вычисленная длина run из vars
fixedLen  = Σ(фиксированных wall) + Σ(ширин окон/дверей из модулей)
fillCount = количество элементов с "fill"
eachFill  = (totalRun - fixedLen) / fillCount
```

Если `eachFill < 0.1` → ошибка: элементы не помещаются в стену.

#### Окно

```jsonc
{ "window": "single" }               // стандартное
{ "window": "double" }               // двустворчатое
{ "window": "wide" }                 // панорамное
{ "window": "single", "y": 1.2 }    // с переопределением высоты от пола
```

Размеры и положение берутся из секции `modules` дескриптора:

| Тип | GLB-модуль | Ширина (w) | Высота проёма (h) | Низ от пола (y) |
|-----|------------|:----------:|:------------------:|:---------------:|
| `single` | `mod_window_single.glb` | 0.90 м | 1.20 м | 0.90 м |
| `double` | `mod_window_double.glb` | 1.40 м | 1.20 м | 0.90 м |
| `wide` | `mod_window_wide.glb` | 2.40 м | 1.50 м | 0.60 м |

- **`w`** — ширина проёма (включая раму), используется при расчёте `fill`
- **`h`** — высота проёма (включая раму)
- **`y`** — расстояние от уровня пола этажа до нижнего края проёма

Поле `"y"` в элементе фасада **переопределяет** значение из `modules`:
```jsonc
{ "window": "single", "y": 1.2 }  // подняли выше стандартного 0.90 м
```

Каждый GLB-модуль окна включает: раму, стекло, подоконник и шторную плоскость.

#### Дверь

```jsonc
{ "door": "entrance" }    // входная дверь
{ "door": "patio" }       // раздвижная/панорамная
```

| Тип | GLB-модуль | Ширина (w) | Высота проёма (h) | Низ от пола (y) |
|-----|------------|:----------:|:------------------:|:---------------:|
| `entrance` | `mod_door_entrance.glb` | 1.00 м | 2.20 м | 0.00 м |
| `patio` | `mod_door_patio.glb` | 2.00 м | 2.30 м | 0.00 м |

Двери всегда начинаются от пола (y = 0).
Каждый GLB-модуль двери включает: раму, полотно/стекло, порог.

## 7. Дополнительные элементы (`features`)

```jsonc
"features": {
  "chimney":  { "position": [0.3, 0.6], "model": "chimney" },
  "gutters":  true,
  "cornice":  true
}
```

| Ключ | Тип | Описание |
|------|-----|----------|
| `chimney` | object \| false | Дымоход. `position` — нормализованные координаты [0..1] на крыше. `model` — ID GLB-модуля |
| `gutters` | boolean | Водосточная система (желоба + трубы по периметру) |
| `cornice` | boolean | Карнизный свес по периметру крыши |

Все элементы `features` опциональны.

### Координаты дымохода

`"position": [0.3, 0.6]` означает:
- 30% длины дома от левого края
- 60% ширины дома от верхнего края (на плане)

Код вычисляет абсолютные координаты и размещает GLB-модуль на поверхности крыши.

## 8. Карта материалов (`materials_map`)

```jsonc
"materials_map": {
  "mat_wall":     { "label": "Стены",              "swappable": true },
  "mat_base":     { "label": "Цоколь",             "swappable": true },
  "mat_roof":     { "label": "Крыша",              "swappable": true },
  "mat_frame":    { "label": "Рамы окон",          "swappable": true },
  "mat_door":     { "label": "Дверь",              "swappable": true },
  "mat_wood":     { "label": "Дерев. отделка",     "swappable": true },
  "mat_metal":    { "label": "Металл (водосток)",  "swappable": true },
  "mat_concrete": { "label": "Бетон (ступени)",    "swappable": true },
  "mat_curtain":  { "label": "Шторы",              "swappable": true }
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| ключ | string | Имя материала в GLB (обязательно начинается с `mat_`) |
| `label` | string | Отображаемое название в UI |
| `swappable` | boolean | Можно ли пользователю менять этот материал |

Код при загрузке GLB-модулей сканирует все материалы с именами `mat_*` и сопоставляет
с этой картой. Материалы с `"swappable": true` показываются в палитре выбора.

Новые типы домов могут вводить дополнительные материалы — достаточно добавить
запись в `materials_map` и использовать соответствующее имя в Blender.

## 9. Примеры планировок

### 9.1. Прямоугольный дом (4 run, 4 turn)

```
┌────────────────┐
│                │
│                │
│                │
└────────────────┘
```

```jsonc
"perimeter": [
  { "run": "L", "facade": "auto_windows", "window_type": "single" },
  { "turn": 90 },
  { "run": "W", "facade": [
      { "wall": "fill" }, { "door": "entrance" }, { "wall": "fill" }
  ]},
  { "turn": 90 },
  { "run": "L", "facade": "auto_windows", "window_type": "single" },
  { "turn": 90 },
  { "run": "W", "facade": "auto_windows", "window_type": "single" },
  { "turn": 90 }
]
```

**Переменные**: `"L": "sqrt(area * 1.6)"`, `"W": "sqrt(area / 1.6)"`

### 9.2. Г-образный (L-shaped) дом (6 run, 6 turn)

```
┌──────────┐
│          │
│          ├────┐
│          │    │
└──────────┴────┘
```

```jsonc
"vars": {
  "L":   "sqrt(area * 1.4)",
  "W":   "sqrt(area / 1.4)",
  "ext": "L * 0.35",
  "ew":  "W * 0.4"
}
```

```jsonc
"perimeter": [
  { "run": "L",   "facade": "auto_windows", "window_type": "single" },
  { "turn": 90 },
  { "run": "W - ew", "facade": [{ "wall": "fill" }] },
  { "turn": -90,  "_comment": "выступ наружу" },
  { "run": "ext", "facade": [
      { "wall": 0.6 }, { "window": "single" }, { "wall": 0.6 }
  ]},
  { "turn": 90 },
  { "run": "ew",  "facade": [
      { "wall": "fill" }, { "door": "entrance" }, { "wall": "fill" }
  ]},
  { "turn": 90 },
  { "run": "L + ext", "facade": "auto_windows", "window_type": "single" },
  { "turn": 90 },
  { "run": "W",  "facade": "auto_windows", "window_type": "single" },
  { "turn": 90 }
]
```

### 9.3. Крестообразный дом (12 run, 12 turn)

```
       ┌─────┐
       │  N  │
  ┌────┤     ├────┐
  │ W  │     │ E  │
  └────┤     ├────┘
       │  S  │
       └─────┘
```

```jsonc
"vars": {
  "core": "sqrt(area * 0.6)",
  "arm":  "sqrt(area * 0.1)"
}
```

```jsonc
"perimeter": [
  { "_comment": "── N верх →" },
  { "run": "core", "facade": [
      { "wall": 0.8 }, { "window": "single" }, { "wall": "fill" },
      { "window": "single" }, { "wall": 0.8 }
  ]},
  { "turn": 90 },

  { "_comment": "── N→E спуск ↓" },
  { "run": "arm", "facade": [{ "wall": "fill" }] },
  { "turn": -90 },

  { "_comment": "── E верх →" },
  { "run": "arm", "facade": [
      { "wall": 0.6 }, { "window": "single" }, { "wall": 0.6 }
  ]},
  { "turn": 90 },

  { "_comment": "── E правый бок ↓" },
  { "run": "core", "facade": [
      { "wall": 0.8 }, { "window": "single" }, { "wall": 0.8 }
  ]},
  { "turn": 90 },
```

## 10. Валидация дескриптора

Код при загрузке JSON проверяет:

| Проверка | Ошибка |
|----------|--------|
| Контур замкнут | `"Контур не замкнут: Δx=..., Δz=..."` |
| Все переменные в `run` определены в `vars` | `"Неизвестная переменная: ..."` |
| `fill` не даёт отрицательных значений | `"Элементы не помещаются в run длиной ..."` |
| Все модули (`window`, `door`) есть в `assets/modules/` | `"Модуль не найден: ..."` |
| `constraints` содержит `min`, `max`, `step`, `default` | `"Неполное ограничение: ..."` |
| `min < default < max` | `"default вне диапазона: ..."` |
| `step > 0` | `"Некорректный шаг: ..."` |
| `materials_map` ключи начинаются с `mat_` | `"Материал без префикса mat_: ..."` |
| Все повороты кратны 90° | `"Поворот не кратен 90°: ..."` (пока поддерживаем только 90°) |
| `floors` не пуст | `"Нет этажей"` |

## 11. Шпаргалка: как создать дескриптор нового дома

1. **Нарисуйте план на бумаге** (вид сверху)

2. **Определите переменные** — какие размеры зависят от площади:
   - Для прямоугольника: `L = sqrt(area * ratio)`, `W = sqrt(area / ratio)`
   - Для сложного плана: выделите основные размеры (core, wing, arm...)

3. **Обойдите периметр** по часовой стрелке от левого верхнего угла:
   - Каждый прямолинейный участок = `run`
   - Каждый поворот = `turn` (+90 внутрь, -90 наружу)

4. **Опишите фасад** каждого run:
   - Расставьте окна и двери
   - Заполните промежутки: `{ "wall": 1.2 }` или `{ "wall": "fill" }`
   - Или используйте `"auto_windows"` для автоматической расстановки

5. **Проверьте замкнутость** — мысленно пройдите весь контур и убедитесь,
   что вернулись в начальную точку

6. **Задайте ограничения** — допустимые диапазоны площади и высоты

7. **Проверьте масштабирование** — мысленно подставьте min и max площади
   и убедитесь, что все `fill` остаются положительными
