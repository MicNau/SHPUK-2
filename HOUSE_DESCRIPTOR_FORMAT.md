# Спецификация формата JSON-дескриптора дома

**Версия формата**: 2.0
**Статус**: согласовано
**Файлы дескрипторов**: `assets/houses/house_type_*.json`

### Изменения v2.0

- Секция `modules`: размеры окон и дверей заданы как диапазоны (`min`/`max`/`default`), а не фиксированные значения
- Добавлены параметрические двери: распашные (1/1.5/2 створки) + раздвижные (1/2 створки)
- Угловые элементы заменены на столб (`pillar`): `pillar_size` в `constraints`
- Добавлена секция `roof_windows` для мансардных (velux) и слуховых (dormer) окон
- Добавлен `modules_override` на уровне этажа для переопределения размеров окон/дверей

---

## 1. Обзор

Каждый тип дома описывается одним JSON-файлом. Файл определяет:

- Метаданные (название, описание)
- Ограничения параметров (допустимые диапазоны площади, высоты и т.д.)
- Тип крыши
- Каталог модулей с диапазонами размеров
- Этажи (один или несколько), каждый со своей планировкой
- Планировку каждого этажа — как **обход периметра по часовой стрелке**
- Мансардные / слуховые окна на крыше
- Дополнительные элементы (дымоход, водостоки, карниз)
- Карту заменяемых материалов

---

## 2. Корневая структура

```jsonc
{
  // ── Метаданные ──────────────────────────────────────
  "id":          "type_a",                              // string, уникальный ID
  "name":        "Одноэтажный дом",                     // string, отображается в UI
  "description": "С вальмовой крышей",                  // string, подпись

  // ── Общие ограничения (для всего дома) ──────────────
  "constraints": {
    "base_h":          { "min": 50, "max": 120, "step": 10, "default": 80, "unit": "cm" },
    "roof_angle":      { "min": 15, "max": 45,  "step": 5,  "default": 25, "unit": "deg" },
    "wall_thickness":  0.20,   // метры, фиксировано
    "pillar_size":     0.20    // метры, сечение углового столба (по умолчанию = wall_thickness)
  },

  // ── Тип крыши ───────────────────────────────────────
  "roof_type": "hip",         // "hip" | "gable" | "flat"

  // ── Каталог модулей (параметры проёмов) ─────────────
  "modules": { /* ... см. раздел 2.1 ... */ },

  // ── Этажи ───────────────────────────────────────────
  "floors": [ /* ... см. раздел 3 ... */ ],

  // ── Мансардные / слуховые окна ──────────────────────
  "roof_windows": [ /* ... см. раздел 7 ... */ ],

  // ── Доп. элементы ───────────────────────────────────
  "features": { /* ... см. раздел 8 ... */ },

  // ── Карта материалов ────────────────────────────────
  "materials_map": { /* ... см. раздел 9 ... */ }
}
```

### Обязательные поля верхнего уровня

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | string | Уникальный идентификатор типа дома |
| `name` | string | Название для отображения в UI |
| `constraints` | object | Общие ограничения (фундамент, крыша, толщина стены, сечение столба) |
| `roof_type` | string | Тип крыши: `"hip"`, `"gable"`, `"flat"` |
| `modules` | object | Каталог модулей с диапазонами размеров (см. раздел 2.1) |
| `floors` | array | Массив этажей (минимум 1) |
| `materials_map` | object | Перечень заменяемых материалов |

### Необязательные поля

| Поле | Тип | Описание |
|------|-----|----------|
| `description` | string | Подпись в UI |
| `roof_windows` | array | Мансардные / слуховые окна (см. раздел 7) |
| `features` | object | Дымоход, водостоки, карниз |

### 2.1. Каталог модулей (`modules`)

Секция `modules` задаёт параметры всех оконных и дверных модулей, используемых
в `perimeter.facade`. Размеры заданы как **диапазоны** с дефолтными значениями.
Код использует эти значения для расчёта `fill`-заполнителей, позиционирования проёмов
и трансформации дочерних объектов GLB.

```jsonc
"modules": {
```

#### Поля модуля окна

| Поле | Тип | Описание |
|------|-----|----------|
| `w` | range | Ширина проёма (включая раму), метры |
| `h` | range | Высота проёма (включая раму), метры |
| `y` | range | Расстояние от уровня пола этажа до нижнего края проёма, метры |
| `frame_profile` | number | Ширина профиля рамы, метры (фиксировано) |
| `sill_overhang` | number | Свес подоконника за раму с каждой стороны, метры |
| `mullions` | array | Список импостов: `"v"` — вертикальный, `"h"` — горизонтальный |
| `glb` | string | Имя GLB-файла в `assets/modules/windows/` |

#### Поля модуля двери

| Поле | Тип | Описание |
|------|-----|----------|
| `w` | range | Ширина проёма (включая раму), метры |
| `h` | range | Высота проёма (включая раму), метры |
| `y` | number | Всегда `0.00` — двери начинаются от пола |
| `leaves` | number | Количество створок: `1`, `1.5`, `2` |
| `mechanism` | string | `"swing"` (распашная) или `"slide"` (раздвижная) |
| `frame_profile` | number | Ширина профиля коробки, метры |
| `glb` | string | Имя GLB-файла в `assets/modules/doors/` |

#### Формат range

```jsonc
{ "min": 0.60, "max": 1.40, "default": 0.90 }
```

- `default` — используется, если в элементе фасада размер не переопределён
- `min` / `max` — для валидации переопределённых значений

#### Ключи модулей и ссылки из фасада

**Ключ** объекта (`window_single`, `door_slide_double` и т.д.) — это ID модуля,
на который ссылаются элементы фасада: `{ "window": "single" }` → ищет `window_single`,
`{ "door": "slide_double" }` → ищет `door_slide_double`.

**Правила формирования ключа**: `<тип>_<вариант>`, где тип = `window` | `door`.
В элементе фасада пишется только вариант: `{ "window": "single" }`, `{ "door": "single" }`.

Новые типы окон/дверей добавляются расширением этой секции:
```jsonc
"window_arch":   { "w": { "min": 0.80, "max": 1.20, "default": 1.00 }, "h": ..., "glb": "mod_window_arch.glb" },
"door_garage":   { "w": { "min": 2.50, "max": 4.00, "default": 3.00 }, "h": ..., "glb": "mod_door_garage.glb" }
```

## 3. Этаж (`floors[i]`)

Каждый элемент массива `floors` описывает один этаж:

```jsonc
{
  "id":       "floor_1",           // string, уникальный ID этажа
  "label":    "Первый этаж",       // string, отображается в UI

  "constraints": {
    "area":    { "min": 40, "max": 100, "step": 5,  "default": 80,  "unit": "m2" },
    "floor_h": { "min": 270, "max": 360, "step": 10, "default": 300, "unit": "cm" }
  },

  "vars": {
    "L": "sqrt(area * 1.6)",
    "W": "sqrt(area / 1.6)"
  },

  // Переопределение размеров окон/дверей для этого этажа (опционально)
  "modules_override": {
    "window_single": { "h": 1.40, "y": 0.80 }
  },

  "perimeter": [ /* ... см. раздел 4 ... */ ]
}
```

### Поля этажа

| Поле | Тип | Обязательное | Описание |
|------|-----|:---:|----------|
| `id` | string | ✓ | Уникальный ID, используется в `S.wallZones` |
| `label` | string | ✓ | Название для UI |
| `constraints` | object | ✓ | Допустимые значения площади и высоты этажа |
| `vars` | object | ✓ | Параметрические переменные (см. раздел 5) |
| `perimeter` | array | ✓ | Обход периметра (см. раздел 4) |
| `modules_override` | object | — | Переопределение размеров окон/дверей для этажа |

### Ограничения этажа (`constraints`)

| Ключ | Что задаёт | Единицы |
|------|-----------|---------|
| `area` | Площадь этажа в плане | м² |
| `floor_h` | Высота этажа (от пола до потолка) | см |

Каждое ограничение — объект:
```jsonc
{ "min": 40, "max": 100, "step": 5, "default": 80, "unit": "m2" }
```

Код конфигуратора:
- Устанавливает `min`, `max`, `step` на элементы ввода
- Округляет значения до шага: `value = round(raw / step) * step`
- Не допускает выход за диапазон

### Переопределение размеров модулей (`modules_override`)

Позволяет задать **другие дефолтные размеры** окон и дверей для конкретного этажа.
Переопределённые значения мержатся с корневыми `modules`:

```jsonc
// Корневой modules:
"window_single": { "w": { ... "default": 0.90 }, "h": { ... "default": 1.20 }, "y": { ... "default": 0.90 } }

// modules_override на 1-м этаже:
"window_single": { "h": 1.40, "y": 0.80 }

// Результат для 1-го этажа:
// w = 0.90 (из корневого), h = 1.40 (переопределён), y = 0.80 (переопределён)
```

Значения в `modules_override` — **числа** (не range-объекты), т.к. это конкретные
дефолты для этажа. Элемент фасада всё ещё может переопределить размер дополнительно.

**Приоритет**: элемент фасада > `modules_override` этажа > корневой `modules.default`

### Многоэтажность

- Этажи собираются **снизу вверх** (первый элемент `floors` = нижний этаж)
- Между этажами автоматически добавляется **перекрытие** толщиной 0.2 м
- Каждый этаж может иметь **свою площадь и пропорции**
- Если верхний этаж меньше нижнего — образуется уступ (пользователь может вручную разметить на нём балкон)
- Верхний этаж **автоматически центрируется** относительно нижнего

## 4. Обход периметра (`perimeter`)

Планировка задаётся как **обход контура по часовой стрелке** (turtle graphics).

### 4.1. Начальные условия

- Стартовая точка: `[0, 0]` (верхний-левый угол на плане)
- Начальное направление: **вправо** (+X)
- Система координат: X — вправо, Z — вниз (вид сверху)

### 4.2. Типы команд

Массив `perimeter` содержит чередующиеся команды двух типов:

#### Команда `run` — движение и рисование стены

```jsonc
{
  "run": "L",              // string — имя переменной из vars (длина участка)
  "facade": [ /* ... */ ]  // array | string — содержимое стены (см. раздел 6)
}
```

#### Команда `turn` — поворот + установка столба

```jsonc
{ "turn": 90 }   // число — угол поворота в градусах
```

| Значение | Направление | Эффект на контуре |
|----------|-------------|-------------------|
| `90` | По часовой стрелке | Обычный угол (внутренний угол дома) |
| `-90` | Против часовой стрелки | Создаёт **выступ** (внешний угол дома) |
| `180` | Разворот | Очень редко, для U-образных элементов |

**Каждый `turn`** автоматически ставит **столб (pillar)** в текущей точке.
Столб — кубоид квадратного сечения (`pillar_size × pillar_size`), универсален
для любого типа угла (внешний, внутренний — без разницы).

### 4.3. Столбы и длина стен

```
Вид сверху:

    стена A    pillar    стена B
  ──────────┐  ┌──┐  ┌──────────
            │  │██│  │
            └──┘  └──┘
```

- На каждый `turn` ставится один `pillar`
- Длина run — это расстояние **от центра столба до центра столба**
- Эффективная длина стены = `run_length - pillar_size`
- Это учитывается автоматически в расчёте `fill`

### 4.4. Правило замкнутости

Контур **обязан быть замкнутым**: после выполнения всех команд текущая точка должна
совпасть с начальной `[0, 0]`. Код проверяет это и выдаёт предупреждение при ошибке.

### 4.5. Порядок команд

Команды `run` и `turn` обычно чередуются:
```
run → turn → run → turn → run → turn → run → turn   (прямоугольник: 4 run + 4 turn)
```

Для сложных планов (L, T, крест) количество команд больше, но принцип тот же.

### 4.6. Комментарии

JSON не поддерживает комментарии, но можно использовать `"_comment"`:
```jsonc
{ "_comment": "── Северная стена →" }
```
Код игнорирует элементы без `run` и без `turn`.

## 5. Параметрические переменные (`vars`)

### 5.1. Формат

```jsonc
"vars": {
  "L":    "sqrt(area * 1.6)",      // длина дома
  "W":    "sqrt(area / 1.6)",      // ширина дома
  "wing": "W * 0.3",              // ширина крыла
  "arm":  "L * 0.25"              // длина крыла
}
```

Каждая переменная — это строка-выражение, которое вычисляется в числовое значение (метры).

### 5.2. Доступные в выражениях величины

| Имя | Откуда | Описание |
|-----|--------|----------|
| `area` | Из UI (constraints этажа) | Площадь этажа, м² |
| Другие переменные из `vars` | Этот же блок | Можно ссылаться на уже определённые |

### 5.3. Поддерживаемые операции

| Операция | Пример | Описание |
|----------|--------|----------|
| `+` `-` `*` `/` | `"area * 1.6"` | Арифметика |
| `sqrt()` | `"sqrt(area * 1.6)"` | Квадратный корень |
| `()` | `"(L + W) / 2"` | Группировка |
| Числовые литералы | `"1.6"`, `"0.3"` | Константы |

### 5.4. Порядок вычисления

Переменные вычисляются **сверху вниз** — каждая может ссылаться только на уже
определённые выше переменные и на `area`.

```jsonc
"vars": {
  "L":    "sqrt(area * 1.6)",    // OK: использует area
  "W":    "sqrt(area / 1.6)",    // OK: использует area
  "wing": "W * 0.3",            // OK: W уже определена
  "diag": "sqrt(L*L + W*W)"     // OK: L и W уже определены
}
```

### 5.5. Когда пересчитываются

Переменные пересчитываются при каждом изменении параметров пользователем
(ползунок площади, поле высоты и т.д.). После пересчёта контур перестраивается.

## 6. Фасад (`facade`)

Описывает содержимое стены на одном `run` — что расположено на этом участке
**слева направо по ходу движения**.

### 6.1. Формат — массив элементов

```jsonc
"facade": [
  { "wall": 1.2 },            // глухая стена 1.2 м
  { "window": "single" },     // одностворчатое окно (размеры из modules)
  { "wall": "fill" },         // заполнитель
  { "window": "single" },     // ещё окно
  { "wall": 1.2 }             // глухая стена 1.2 м
]
```

### 6.2. Формат — автоматические окна

```jsonc
"facade": "auto_windows",     // строка вместо массива
"window_type": "single",      // какой тип окна расставлять
"min_margin": 1.0             // мин. отступ от краёв и между окнами (по умолчанию 1.0)
```

Код рассчитает:
```
effectiveLen = runLength - pillar_size
winW = modules[window_type].w.default
winCount = floor(effectiveLen / (winW + min_margin * 2))
margin = (effectiveLen - winCount * winW) / (winCount + 1)
```

### 6.3. Типы элементов фасада

#### Глухая стена

```jsonc
{ "wall": 2.0 }       // фиксированная длина, метры
{ "wall": "fill" }     // заполнитель (вычисляется)
```

**`"fill"`** — оставшееся после размещения фиксированных элементов место делится
**поровну** между всеми `"fill"` в этом `run`.

Алгоритм:
```
effectiveRun = вычисленная длина run - pillar_size  // стена между столбами
fixedLen     = Σ(фиксированных wall) + Σ(ширин окон/дверей из модулей)
fillCount    = количество элементов с "fill"
eachFill     = (effectiveRun - fixedLen) / fillCount
```

Если `eachFill < 0.1` → ошибка: элементы не помещаются в стену.

#### Окно

```jsonc
{ "window": "single" }                                 // дефолтные размеры из modules
{ "window": "double" }                                 // двустворчатое
{ "window": "wide" }                                   // панорамное
{ "window": "single", "w": 1.10 }                     // переопределена ширина
{ "window": "single", "h": 1.40 }                     // переопределена высота
{ "window": "single", "y": 1.2 }                      // переопределена высота подоконника
{ "window": "single", "w": 1.10, "h": 1.40, "y": 0.80 }  // всё переопределено
{ "window": "single", "w": 0.60, "h": 0.50, "y": 1.50 }  // маленькое окно (санузел)
```

**Приоритет размеров**: элемент фасада > `modules_override` этажа > `modules.*.default`

Размеры и параметры из секции `modules`:

| Поле | Описание |
|------|----------|
| `w` | Ширина проёма (включая раму), используется при расчёте `fill` |
| `h` | Высота проёма (включая раму) |
| `y` | Расстояние от уровня пола этажа до нижнего края проёма |
| `frame_profile` | Ширина профиля рамы (для трансформации GLB) |
| `sill_overhang` | Свес подоконника за раму (для трансформации GLB) |

Каждый GLB-модуль окна включает именованные дочерние объекты: `frame_left`,
`frame_right`, `frame_top`, `frame_bottom`, `sill`, `glass`, `curtain`
(и `mullion_v`, `mullion_h` для двустворчатых/панорамных). Код трансформирует их
для достижения целевого размера (см. `HOUSE_MODULES_SPEC.md`, раздел 2.5).

#### Дверь

```jsonc
{ "door": "single" }                       // распашная одностворчатая
{ "door": "onehalf" }                      // распашная полуторастворчатая
{ "door": "double" }                       // распашная двустворчатая
{ "door": "slide_single" }                 // раздвижная одностворчатая
{ "door": "slide_double" }                 // раздвижная двустворчатая

{ "door": "slide_double", "w": 2.40 }     // широкая раздвижная
```

| Тип | Створки | Механизм | Дефолт w | Дефолт h |
|-----|:-------:|----------|:--------:|:--------:|
| `single` | 1 | распашная | 0.90 м | 2.10 м |
| `onehalf` | 1.5 | распашная | 1.20 м | 2.10 м |
| `double` | 2 | распашная | 1.40 м | 2.10 м |
| `slide_single` | 1 | раздвижная | 1.00 м | 2.10 м |

Каждый GLB-модуль двери включает именованные дочерние объекты: `frame_left`,
`frame_right`, `frame_top`, `threshold`, `leaf_main`, `handle`
(и `leaf_minor` для полуторастворчатых/двустворчатых, `rail_top`, `rail_bottom` для раздвижных).

## 7. Мансардные и слуховые окна (`roof_windows`)

Секция `roof_windows` задаёт окна, размещаемые на скатах крыши.
Два типа: **velux** (в плоскости ската) и **dormer** (люкарна, выступает из ската).

```jsonc
"roof_windows": [
  {
    "module": "window_velux",
    "w": 0.78,
    "h": 0.98,
    "slope": "south",
    "position_along": 0.5,
    "position_up": 0.5,
    "count": 2,
    "spacing": 1.5
  },
  {
    "module": "dormer",
    "w": 1.50,
    "h": 1.40,
    "depth": 1.00,
    "slope": "south",
    "position_along": 0.3,
    "position_up": 0.3,
    "window": {
      "type": "single",
      "w": 0.90,
      "h": 1.00
    }
  }
]
```

### 7.1. Общие поля

| Поле | Тип | Обязательное | Описание |
|------|-----|:---:|----------|
| `module` | string | ✓ | ID модуля: `"window_velux"` или `"dormer"` |
| `slope` | string | ✓ | На каком скате: `"north"`, `"south"`, `"east"`, `"west"` |
| `position_along` | number | ✓ | Позиция вдоль конька (0.0 = левый край, 1.0 = правый край) |
| `position_up` | number | ✓ | Позиция вверх по скату (0.0 = карниз, 1.0 = конёк) |
| `count` | number | — | Количество окон в ряд (по умолчанию 1) |
| `spacing` | number | — | Расстояние между центрами окон, метры (нужен при `count > 1`) |

### 7.2. Velux (мансардное окно в плоскости ската)

| Поле | Тип | Описание |
|------|-----|----------|
| `w` | number | Ширина окна, метры |
| `h` | number | Высота окна (вдоль ската), метры |

Значения `w` и `h` должны быть в пределах диапазонов из `modules.window_velux`.

Код при установке:
1. Вычисляет 3D-позицию на скате по `position_along` и `position_up`
2. Вырезает прямоугольник в геометрии ската
3. Клонирует и трансформирует GLB-модуль
4. Размещает в плоскости ската

### 7.3. Dormer (слуховое окно / люкарна)

| Поле | Тип | Описание |
|------|-----|----------|
| `w` | number | Ширина фронта люкарны, метры |
| `h` | number | Высота стенки люкарны, метры |
| `depth` | number | Глубина выступа от ската, метры |
| `window` | object | Параметры встроенного окна (см. ниже) |

**Встроенное окно** (`window`):

```jsonc
"window": {
  "type": "single",    // тип окна (single, double)
  "w": 0.90,           // ширина (должна быть < w люкарны)
  "h": 1.00            // высота (должна быть < h люкарны)
}
```

Код при установке:
1. Вычисляет 3D-позицию основания на скате
2. Вырезает область в геометрии ската
3. Клонирует и трансформирует GLB-модуль dormer (стенки, мини-крыша)
4. Трансформирует встроенное окно внутри dormer
5. Размещает конструкцию, ориентируя фронт наружу

## 8. Дополнительные элементы (`features`)

```jsonc
"features": {
  "chimney":  { "position": [0.3, 0.6], "model": "chimney" },
  "gutters":  true,
  "cornice":  true
}
```

| Ключ | Тип | Описание |
|------|-----|----------|
| `chimney` | object \| false | Дымоход. `position` — нормализованные координаты [0..1] на крыше. `model` — ID GLB-модуля |
| `gutters` | boolean | Водосточная система (желоба + трубы по периметру) |
| `cornice` | boolean | Карнизный свес по периметру крыши |

Все элементы `features` опциональны.

### Координаты дымохода

`"position": [0.3, 0.6]` означает:
- 30% длины дома от левого края
- 60% ширины дома от верхнего края (на плане)

Код вычисляет абсолютные координаты и размещает GLB-модуль на поверхности крыши.

## 11. Валидация дескриптора

Код при загрузке JSON проверяет:

| Проверка | Ошибка |
|----------|--------|
| Контур замкнут | `"Контур не замкнут: Δx=..., Δz=..."` |
| Все переменные в `run` определены в `vars` | `"Неизвестная переменная: ..."` |
| `fill` не даёт отрицательных значений | `"Элементы не помещаются в run длиной ..."` |
| Все модули (`window`, `door`) есть в `modules` | `"Модуль не найден: ..."` |
| Переопределённые размеры в диапазоне `min..max` | `"Размер вне диапазона: window_single.w = 2.0, max = 1.40"` |
| `constraints` содержит `min`, `max`, `step`, `default` | `"Неполное ограничение: ..."` |
| `min < default < max` | `"default вне диапазона: ..."` |
| `step > 0` | `"Некорректный шаг: ..."` |
| `materials_map` ключи начинаются с `mat_` | `"Материал без префикса mat_: ..."` |
| Все повороты кратны 90° | `"Поворот не кратен 90°: ..."` (пока поддерживаем только 90°) |
| `floors` не пуст | `"Нет этажей"` |
| `roof_windows` — `w`, `h` в допустимых диапазонах | `"Размер мансардного окна вне диапазона: ..."` |
| `roof_windows` — `position_along` и `position_up` в [0, 1] | `"Позиция окна на крыше вне диапазона: ..."` |
| `dormer.window.w < dormer.w` | `"Окно люкарны шире самой люкарны: ..."` |
| `dormer.window.h < dormer.h` | `"Окно люкарны выше стенки люкарны: ..."` |

## 12. Шпаргалка: как создать дескриптор нового дома

1. **Нарисуйте план на бумаге** (вид сверху)

2. **Определите переменные** — какие размеры зависят от площади:
   - Для прямоугольника: `L = sqrt(area * ratio)`, `W = sqrt(area / ratio)`
   - Для сложного плана: выделите основные размеры (core, wing, arm...)

3. **Обойдите периметр** по часовой стрелке от левого верхнего угла:
   - Каждый прямолинейный участок = `run`
   - Каждый поворот = `turn` (+90 внутрь, -90 наружу)
   - Помните: каждый `turn` = столб, длина стены = run - pillar_size

4. **Настройте модули** — задайте диапазоны размеров для окон и дверей:
   - Разные размеры на разных этажах → `modules_override`
   - Уникальные окна (санузел, кладовая) → переопределение в элементе фасада

5. **Опишите фасад** каждого run:
   - Расставьте окна и двери
   - Заполните промежутки: `{ "wall": 1.2 }` или `{ "wall": "fill" }`
   - Или используйте `"auto_windows"` для автоматической расстановки

6. **Добавьте мансардные окна** (если крыша не плоская):
   - Velux — для функциональных окон (свет и вентиляция)
   - Dormer — для декоративных / жилых мансардных окон

7. **Проверьте замкнутость** — мысленно пройдите весь контур и убедитесь,
   что вернулись в начальную точку

8. **Задайте ограничения** — допустимые диапазоны площади и высоты

9. **Проверьте масштабирование** — мысленно подставьте min и max площади
   и убедитесь, что все `fill` остаются положительными
